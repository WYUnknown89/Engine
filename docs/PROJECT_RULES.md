# Project Rules

## Purpose

These rules govern every Codex session.

## Golden Rules

-   The canonical specification is
    `docs/Master_Implementation_Specification.md`.
-   Never change architecture without approval.
-   Never weaken performance, determinism or testing requirements.
-   Never add third-party dependencies without approval.
-   Never delete tests to make a build pass.
-   Never claim incomplete functionality is finished.

## Workflow

1.  Inspect
2.  Plan
3.  Implement
4.  Build
5.  Test
6.  Verify
7.  Report

## Milestone Rules

-   Only work on the approved milestone.
-   Do not start the next milestone automatically.
-   Stop only for genuine blockers.
